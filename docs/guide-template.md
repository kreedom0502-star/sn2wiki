# 攻略文章模板

用于 `guides/*.html` 的首批 SEO 内容。每篇页面都应该先回答玩家搜索意图，再说明资料状态，避免把待实测内容写成确定结论。

## 页面结构

```text
H1：Subnautica 2 + 具体问题

适用版本：
最后验证：
资料状态：
搜索意图：

快速答案：
用 2-4 句话直接回答玩家问题。未实测时明确写“待实测”，不要写伪坐标。

当前可用结论：
列出已经能确认的范围，比如用途、前置关系、需要复核的项目。

实测路线：
按出生点、地标、深度、风险、装备要求组织。没有实测就写待补充清单。

相关资料：
链接到资源、蓝图、配方、地图和更新日志。

更新记录：
记录创建日期、复核日期和受影响版本。
```

## 元信息建议

- `适用版本`：`Early Access 待复核`、`EA 1.0 已验证`。
- `最后验证`：没有游戏内验证时写 `待游戏内实测`。
- `资料状态`：`占位模板`、`待实测`、`已验证`、`待复核`。
- `搜索意图`：如 `资源位置`、`载具解锁`、`联机问题`。

## 写作规则

- 不写未经验证的具体坐标、刷新机制或材料数量。
- 能确认的事实和待验证内容分开写。
- 资源、蓝图、配方、地图点位互相链接。
- 官方更新后，先检查受影响页面的版本状态。
